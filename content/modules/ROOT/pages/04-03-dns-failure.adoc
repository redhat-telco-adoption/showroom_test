= Lab 4.3: DNS Resolution Failure
include::_attributes.adoc[]

[.label.label-red]#Advanced# | *Persona:* Platform Engineer | *Category:* Networking & Routing

== Overview

The `lab-4-3-frontend` application displays a page but shows an error: "Cannot connect to backend service." The frontend tries to reach `lab-4-3-backend-svc` but DNS resolution fails due to a NetworkPolicy that blocks egress UDP port 53 (DNS). Your mission is to identify the DNS blocking and fix the NetworkPolicy.

== Hints

[%collapsible]
.Hint 1 -- Where to look
====
DNS uses UDP port 53. If a NetworkPolicy restricts egress traffic and does not explicitly allow UDP port 53, DNS queries from the pod will time out. Check the egress rules of the NetworkPolicy applied to the frontend pods.
====

[%collapsible]
.Hint 2 -- What needs to change
====
The NetworkPolicy `lab-4-3-frontend-egress` has an egress rule allowing TCP traffic but not UDP. You need to add an egress rule that allows UDP port 53 to the cluster DNS service.
====

[%collapsible]
.Hint 3 -- Almost there
====
Edit the NetworkPolicy to add an egress rule allowing UDP on port 53. The rule should allow traffic to the `openshift-dns` namespace or simply allow all egress UDP/53 traffic.
====

== Fix the Issue

Update the NetworkPolicy to allow DNS egress traffic (UDP port 53) from the frontend pods.

[%collapsible]
.Reveal Solution (Instructor Reference Only)
====
Edit the NetworkPolicy:

[.console-input]
[source,bash,subs="attributes+"]
----
oc edit networkpolicy lab-4-3-frontend-egress -n {user}-lab
----

Add the following egress rule to the existing `egress` list:

[source,yaml]
----
- ports:
  - protocol: UDP
    port: 53
----

Or patch it:

[.console-input]
[source,bash,subs="attributes+"]
----
cat <<'EOF' | oc apply -n {user}-lab -f -
apiVersion: networking.k8s.io/v1
kind: NetworkPolicy
metadata:
  name: lab-4-3-frontend-egress
spec:
  podSelector:
    matchLabels:
      app: lab-4-3-frontend
  policyTypes:
  - Egress
  egress:
  - ports:
    - protocol: TCP
      port: 8080
  - ports:
    - protocol: UDP
      port: 53
EOF
----
====
