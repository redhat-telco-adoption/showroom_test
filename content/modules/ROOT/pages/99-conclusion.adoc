Congratulations! You have completed the OpenShift Troubleshooting Workshop. You worked through 19 realistic production failures, diagnosing and fixing each one using the `oc` CLI and the OpenShift Web Console.

== What you've learned

You have practiced troubleshooting skills across 6 categories:

* *Configuration* -- identifying and resolving missing ConfigMaps, missing Secrets, and invalid environment variable configurations
* *Storage* -- diagnosing missing PVCs, incorrect StorageClasses, and volume permission errors caused by `fsGroup` mismatches
* *Security* -- resolving RBAC permission denials, SCC violations blocking privileged containers, and missing image pull Secrets
* *Networking* -- fixing Service selector mismatches, Route `targetPort` errors, DNS egress blocking, and NetworkPolicy ingress restrictions
* *Resources* -- correcting over-allocated resource requests, OOMKilled memory limits, and ResourceQuota exhaustion
* *Application lifecycle* -- removing bad command overrides, fixing misconfigured readiness and liveness probes, and correcting invalid image tags

== Key troubleshooting patterns

Across all labs you applied a consistent diagnostic approach:

. *Observe symptoms* -- check pod status, events, and logs to understand what is failing
. *Identify root cause* -- use `oc describe`, `oc get events`, and YAML inspection to find the specific misconfiguration
. *Apply a targeted fix* -- change only what is needed to resolve the root cause
. *Verify the outcome* -- confirm the pod reaches `Running` state and the Route serves traffic

This pattern applies to the vast majority of OpenShift application failures you will encounter in production.
