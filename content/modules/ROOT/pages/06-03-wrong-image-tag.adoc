= Lab 6.3: Wrong Image Tag
include::_attributes.adoc[]
:module_namespace: {ns_lifecycle}

[.label.label-green]#Beginner# | *Persona:* Developer | *Category:* Application Lifecycle

== Overview

The application `lab-6-3-app` cannot start because the container image tag does not exist. The Deployment references a non-existent image tag, causing an `ImagePullBackOff` error. Your mission is to identify the bad image reference and correct it to a valid tag.

== Learning objectives

* Identify `ImagePullBackOff` caused by a non-existent image tag using `oc get events` and image inspection
* Understand how image pull failures block container startup when registry manifests are not found
* Apply `oc set image` to update the Deployment with a valid image tag and restore the application to `Running` state

=== Diagnose

[tabs]
====
Web Console::
+
--
. Navigate to *Workloads -> Pods* in namespace `{module_namespace}`
. Click the `lab-6-3-app` pod
. Review the *Events* tab -- note the "manifest unknown" error
. This means the image tag `v99.99` does not exist in the registry
. Navigate to *Workloads -> Deployments*, click `lab-6-3-app`, and check the image reference
--

Web Terminal (`oc`)::
+
--
Check the image reference:

[.console-input]
[source,bash,subs="attributes+"]
----
oc get deployment lab-6-3-app -n {module_namespace} -o jsonpath='{.spec.template.spec.containers[0].image}'
----

[.console-output]
[source,text]
----
registry.access.redhat.com/ubi8/httpd-24:v99.99
----

The tag `v99.99` does not exist. You need to use a valid tag like `latest`.
--
====

=== Hints

[%collapsible]
.Hint 1 -- Where to look
====
The `ImagePullBackOff` error means the container runtime cannot pull the specified image. This usually means the image or tag does not exist, or you lack credentials (but this image is public).
====

[%collapsible]
.Hint 2 -- What's wrong
====
The image tag `v99.99` does not exist for the `ubi8/httpd-24` image. The image is publicly available at `registry.access.redhat.com/ubi8/httpd-24` but you need a valid tag.
====

[%collapsible]
.Hint 3 -- Almost there
====
Change the image tag from `v99.99` to `latest`. Use `oc set image` to update the Deployment.
====

== Exercise 2: Resolve the issue

=== Fix the issue

Update the container image reference to use a valid tag that exists in the registry.

[%collapsible]
.Reveal Solution (Instructor Reference Only)
====
[.console-input]
[source,bash,subs="attributes+"]
----
oc set image deployment/lab-6-3-app \
  lab-6-3-app=registry.access.redhat.com/ubi8/httpd-24:latest \
  -n {module_namespace}
----

This triggers a new rollout with the correct image tag.
====
