== Overview

The application `lab-3-3-app` cannot start because the container image cannot be pulled. The image is hosted in a private registry that requires authentication. Your mission is to create the appropriate image pull secret and configure the Deployment to use it.

== Hints

[%collapsible]
.Hint 1 -- Where to look
====
The image is from a private registry that requires authentication. You need to create a docker-registry type Secret with the correct credentials and link it to the pod.
====

[%collapsible]
.Hint 2 -- What to create
====
Use `oc create secret docker-registry` to create a Secret with the registry URL, username, and password. Your instructor will provide the credentials for the private registry on the lab instructions board.
====

[%collapsible]
.Hint 3 -- Almost there
====
After creating the Secret, you can either:

1. Add it to the Deployment's `spec.template.spec.imagePullSecrets`
2. Link it to the default ServiceAccount with `oc secrets link`

Then delete the pod to trigger a re-pull.
====

== Fix the Issue

Create a docker-registry Secret with the credentials provided by your instructor, and configure the Deployment or ServiceAccount to use it. Your instructor will provide the registry credentials.

[%collapsible]
.Reveal Solution (Instructor Reference Only)
====
Create the image pull Secret (instructor provides actual credentials):

[.console-input]
[source,bash,subs="attributes+"]
----
oc create secret docker-registry registry-creds \
  --docker-server=quay.io \
  --docker-username=workshop-robot \
  --docker-password=INSTRUCTOR_PROVIDED_TOKEN \
  -n {module_namespace}
----

Link the Secret to the default ServiceAccount:

[.console-input]
[source,bash,subs="attributes+"]
----
oc secrets link default registry-creds --for=pull -n {module_namespace}
----

Delete the pod to trigger a re-pull:

[.console-input]
[source,bash,subs="attributes+"]
----
oc delete pod -l app=lab-3-3-app -n {module_namespace}
----
====
