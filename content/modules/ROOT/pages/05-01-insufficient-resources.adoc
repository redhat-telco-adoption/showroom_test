== Overview

The application `lab-5-1-app` is stuck in `Pending` state. The pod cannot be scheduled because it requests far more CPU and memory than any node in the cluster can provide. Your mission is to reduce the resource requests to reasonable values so the pod can be scheduled.

== Hints

[%collapsible]
.Hint 1 -- Where to look
====
The pod requests `cpu: "8"` and `memory: "32Gi"`. Check the cluster node capacity with `oc adm top nodes` to see how much is actually available.
====

[%collapsible]
.Hint 2 -- What to change
====
The nginx application needs very little resources. Reasonable requests would be `cpu: "100m"` and `memory: "128Mi"`. The current values are vastly oversized.
====

[%collapsible]
.Hint 3 -- Almost there
====
Use `oc set resources` or `oc edit deployment` to update the resource requests and limits to reasonable values.
====

== Fix the Issue

Reduce the resource requests and limits in the Deployment to values that can be satisfied by the cluster nodes.

[%collapsible]
.Reveal Solution (Instructor Reference Only)
====
[.console-input]
[source,bash,subs="attributes+"]
----
oc set resources deployment/lab-5-1-app \
  --requests=cpu=100m,memory=128Mi \
  --limits=cpu=200m,memory=256Mi \
  -n {module_namespace}
----

This triggers a new rollout with the reduced resource requirements. The scheduler will find a suitable node.
====
