= oc Cheat Sheet
include::_attributes.adoc[]

A quick reference of commonly used `oc` commands for troubleshooting OpenShift applications.

== Pods

[cols="2,3", options="header"]
|===
| Command | Description

| `oc get pods -n <ns>`
| List all pods in a namespace

| `oc get pods -n <ns> -o wide`
| List pods with node and IP details

| `oc describe pod <name> -n <ns>`
| Detailed pod info including events

| `oc logs <pod> -n <ns>`
| View container logs

| `oc logs <pod> -n <ns> --previous`
| View logs from previous crashed container

| `oc logs -l app=<name> -n <ns>`
| View logs by label selector

| `oc debug deployment/<name> -n <ns>`
| Start a debug pod based on a Deployment

| `oc rsh deployment/<name> -n <ns>`
| Open a remote shell in a running pod

| `oc exec <pod> -n <ns> -- <command>`
| Execute a command in a running container

| `oc delete pod <name> -n <ns>`
| Delete a pod (triggers recreation by Deployment)
|===

== Events

[cols="2,3", options="header"]
|===
| Command | Description

| `oc get events -n <ns>`
| List events in a namespace

| `oc get events -n <ns> --sort-by='.lastTimestamp'`
| List events sorted by time

| `oc get events -n <ns> --field-selector type=Warning`
| Show only warning events

| `oc describe pod <name> -n <ns>`
| Show events specific to a pod
|===

== ConfigMaps & Secrets

[cols="2,3", options="header"]
|===
| Command | Description

| `oc get configmap -n <ns>`
| List ConfigMaps

| `oc describe configmap <name> -n <ns>`
| Show ConfigMap details and data

| `oc create configmap <name> --from-literal=key=value -n <ns>`
| Create a ConfigMap from literal values

| `oc get secret -n <ns>`
| List Secrets

| `oc describe secret <name> -n <ns>`
| Show Secret metadata (not values)

| `oc create secret generic <name> --from-literal=key=value -n <ns>`
| Create a generic Secret

| `oc create secret docker-registry <name> --docker-server=<url> --docker-username=<user> --docker-password=<pass> -n <ns>`
| Create an image pull Secret

| `oc set env deployment/<name> KEY=VALUE -n <ns>`
| Set environment variables on a Deployment
|===

== Storage (PVC / PV)

[cols="2,3", options="header"]
|===
| Command | Description

| `oc get pvc -n <ns>`
| List PersistentVolumeClaims

| `oc describe pvc <name> -n <ns>`
| Show PVC details and events

| `oc get pv`
| List PersistentVolumes (cluster-wide)

| `oc get storageclass`
| List available StorageClasses

| `oc delete pvc <name> -n <ns>`
| Delete a PVC
|===

== Networking

[cols="2,3", options="header"]
|===
| Command | Description

| `oc get svc -n <ns>`
| List Services

| `oc describe svc <name> -n <ns>`
| Show Service details including selector and ports

| `oc get endpoints <name> -n <ns>`
| Check Service endpoints (should show pod IPs)

| `oc get route -n <ns>`
| List Routes

| `oc describe route <name> -n <ns>`
| Show Route details including target port

| `oc get networkpolicy -n <ns>`
| List NetworkPolicies

| `oc describe networkpolicy <name> -n <ns>`
| Show NetworkPolicy rules

| `oc patch svc <name> -n <ns> --type=json -p='[...]'`
| Patch a Service (e.g., fix selector)

| `oc patch route <name> -n <ns> --type=json -p='[...]'`
| Patch a Route (e.g., fix targetPort)
|===

== RBAC

[cols="2,3", options="header"]
|===
| Command | Description

| `oc auth can-i <verb> <resource> -n <ns>`
| Check if you can perform an action

| `oc auth can-i <verb> <resource> --as=system:serviceaccount:<ns>:<sa> -n <ns>`
| Check if a ServiceAccount can perform an action

| `oc get rolebinding -n <ns>`
| List RoleBindings in a namespace

| `oc get clusterrolebinding`
| List ClusterRoleBindings

| `oc create role <name> --verb=<verbs> --resource=<resources> -n <ns>`
| Create a Role

| `oc create rolebinding <name> --role=<role> --serviceaccount=<ns>:<sa> -n <ns>`
| Create a RoleBinding

| `oc adm policy add-scc-to-user <scc> -z <sa> -n <ns>`
| Grant an SCC to a ServiceAccount

| `oc get scc`
| List Security Context Constraints
|===

== Resources & scheduling

[cols="2,3", options="header"]
|===
| Command | Description

| `oc adm top nodes`
| Show node resource usage

| `oc adm top pods -n <ns>`
| Show pod resource usage

| `oc get resourcequota -n <ns>`
| List ResourceQuotas

| `oc describe resourcequota <name> -n <ns>`
| Show quota usage and limits

| `oc set resources deployment/<name> --requests=cpu=100m,memory=128Mi -n <ns>`
| Set resource requests

| `oc set resources deployment/<name> --limits=cpu=200m,memory=256Mi -n <ns>`
| Set resource limits

| `oc scale deployment/<name> --replicas=<n> -n <ns>`
| Scale a Deployment
|===

== Deployments & rollouts

[cols="2,3", options="header"]
|===
| Command | Description

| `oc get deployment -n <ns>`
| List Deployments

| `oc describe deployment <name> -n <ns>`
| Show Deployment details and events

| `oc rollout status deployment/<name> -n <ns>`
| Watch rollout progress

| `oc rollout restart deployment/<name> -n <ns>`
| Trigger a new rollout

| `oc set image deployment/<name> <container>=<image> -n <ns>`
| Update the container image

| `oc edit deployment <name> -n <ns>`
| Edit a Deployment in your default editor
|===

== Useful flags

[cols="2,3", options="header"]
|===
| Flag | Description

| `-o yaml`
| Output in YAML format

| `-o json`
| Output in JSON format

| `-o jsonpath='{.spec.field}'`
| Extract specific fields

| `-o wide`
| Show additional columns

| `--show-labels`
| Show resource labels

| `-l app=<name>`
| Filter by label selector

| `--all-namespaces` or `-A`
| Search across all namespaces

| `-w` or `--watch`
| Watch for changes in real time
|===
