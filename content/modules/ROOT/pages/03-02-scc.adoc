== Overview

The application `lab-3-2-app` in the `{module_namespace}` namespace cannot start because its Deployment spec requests to run as root (UID 0). OpenShift's Security Context Constraints (SCCs) prevent this by default. Your mission is to understand why the pod is rejected and fix the issue.

== Learning objectives

* Identify SCC admission errors blocking pod creation due to a `runAsUser: 0` requirement
* Understand how Security Context Constraints (SCCs) restrict privileged container configurations in OpenShift
* Apply a Deployment patch to remove the root user requirement and comply with the default `restricted-v2` SCC

=== Hints

[%collapsible]
.Hint 1 -- Where to look
====
OpenShift uses Security Context Constraints (SCCs) to control what security settings pods can use. The default `restricted-v2` SCC does not allow running as root (UID 0). Check the Deployment's `securityContext` settings.
====

[%collapsible]
.Hint 2 -- 2 ways to fix this
====
There are 2 ways to fix this:

1. *Recommended:* Remove the `runAsUser: 0` from the Deployment spec. The nginx image works fine without root.
2. *Alternative:* Grant the `anyuid` SCC to the ServiceAccount. This is less secure and should only be used when the application truly requires root.
====

[%collapsible]
.Hint 3 -- Almost there
====
The simplest fix is to edit the Deployment and remove the `securityContext.runAsUser: 0` field. Use `oc edit deployment` or `oc patch`.
====

== Exercise 2: Resolve the issue

=== Fix the issue

Remove the root user requirement from the Deployment spec so it complies with OpenShift's default SCC, or grant the appropriate SCC to the ServiceAccount.

[%collapsible]
.Reveal Solution (Instructor Reference Only)
====
*Option A (Recommended) -- Remove the root requirement:*

[.console-input]
[source,bash,subs="attributes+"]
----
oc patch deployment lab-3-2-app -n {module_namespace} --type=json \
  -p='[{"op": "remove", "path": "/spec/template/spec/containers/0/securityContext/runAsUser"}]'
----

*Option B -- Grant anyuid SCC (less secure):*

[.console-input]
[source,bash,subs="attributes+"]
----
oc adm policy add-scc-to-user anyuid -z default -n {module_namespace}
----

Then restart the pod:

[.console-input]
[source,bash,subs="attributes+"]
----
oc delete pod -l app=lab-3-2-app -n {module_namespace}
----
====
