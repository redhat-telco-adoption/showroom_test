Welcome to the OpenShift Troubleshooting Workshop!

In this workshop, you take on the role of a senior SRE at ACME Corp, tasked with investigating and fixing a series of broken application deployments on OpenShift. Each module presents a realistic failure scenario, and your job is to diagnose the root cause and restore the application to a running state.

== What you'll learn

In this workshop, you will:

* Use `oc` CLI and the OpenShift web console to investigate pod failures
* Identify and fix configuration errors involving ConfigMaps, Secrets, and environment variables
* Resolve storage, RBAC, and networking issues that prevent applications from starting
* Interpret pod events, logs, and status conditions to find root causes quickly

== Who this is for

This workshop is designed for developers and SREs who work with OpenShift and want to build practical troubleshooting skills.

**Experience level**: All levels welcome. Module 1 provides foundational skills, and each module builds on the previous one. Experienced practitioners can start at any module.

== Prerequisites

Before starting this workshop, you should have:

* Basic familiarity with Linux command line
* A web browser and internet connection
* Access to your workshop environment (credentials provided in the lab interface)

No prior OpenShift experience is required for Module 1.
