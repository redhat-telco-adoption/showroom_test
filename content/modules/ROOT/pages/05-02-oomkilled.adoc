== Overview

The application `lab-5-2-app` keeps crashing with exit code 137. The container is being killed by the kernel's Out-Of-Memory (OOM) killer because the memory limit is set too low for the application's needs. Your mission is to identify the OOMKilled condition and increase the memory limit.

== Hints

[%collapsible]
.Hint 1 -- Where to look
====
Exit code 137 means the process was killed by SIGKILL, which is what happens during an OOM kill. The container's memory limit is too restrictive for the application's runtime needs.
====

[%collapsible]
.Hint 2 -- What's wrong
====
The memory limit is `32Mi` but the application needs approximately 100MB at runtime. The OOM killer terminates the container when it exceeds the limit.
====

[%collapsible]
.Hint 3 -- Almost there
====
Increase the memory limit to at least `256Mi`. Use `oc set resources` to update both requests and limits.
====

== Fix the Issue

Increase the memory limit for the Deployment so the application has enough memory to run without being OOM killed.

[%collapsible]
.Reveal Solution (Instructor Reference Only)
====
[.console-input]
[source,bash,subs="attributes+"]
----
oc set resources deployment/lab-5-2-app \
  --requests=memory=128Mi \
  --limits=memory=256Mi \
  -n {module_namespace}
----

This triggers a new rollout with sufficient memory allocation.
====
