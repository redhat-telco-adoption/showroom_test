= Lab 1.1: Missing ConfigMap

== Overview

The application `lab-1-1-app` is deployed in your namespace but the pod is stuck in an error state. The container cannot start because it references a ConfigMap that does not exist. Your mission is to identify the missing ConfigMap and create it so the application starts successfully.

== Hints

[%collapsible]
.Hint 1 -- Where to look
====
The pod is trying to mount a ConfigMap as a volume. Check the Deployment spec to see what volume mounts are expected and what ConfigMap name is referenced.

[.console-input]
[source,bash,subs="attributes+"]
----
oc get deployment lab-1-1-app -n {module_namespace} -o yaml | grep -A 10 volumes
----
====

[%collapsible]
.Hint 2 -- What's missing
====
The Deployment mounts a ConfigMap named `app-config` at `/usr/share/nginx/html`. This ConfigMap should contain a key named `index.html` that provides the web page content.
====

[%collapsible]
.Hint 3 -- Almost there
====
Create a ConfigMap with the name `app-config` and a key `index.html`. The value should be valid HTML content. Use `oc create configmap` or apply a YAML manifest.
====

== Fix the Issue

You need to create the missing ConfigMap named `app-config` in the `{module_namespace}` namespace. The ConfigMap must contain a key called `index.html` with HTML content that the nginx container will serve.

[%collapsible]
.Reveal Solution (Instructor Reference Only)
====
[.console-input]
[source,bash,subs="attributes+"]
----
cat <<'EOF' | oc apply -n {module_namespace} -f -
apiVersion: v1
kind: ConfigMap
metadata:
  name: app-config
data:
  index.html: |
    <!DOCTYPE html>
    <html>
    <head><title>Lab 1.1</title></head>
    <body>
      <h1>Congratulations! You fixed the Missing ConfigMap issue.</h1>
    </body>
    </html>
EOF
----

After creating the ConfigMap, the pod will automatically restart and transition to `Running`.
====
