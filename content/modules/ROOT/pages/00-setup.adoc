= Environment Setup
include::_attributes.adoc[]

Before starting the labs, verify that your environment is set up correctly.

== Step 1: Enter Your Workshop Credentials

Look for the red credential bar at the top of this page. Enter:

* *Username* -- your assigned username (e.g. `user1`)
* *Cluster subdomain* -- provided by your instructor (e.g. `apps.cluster-xxxx.example.com`)

Then click *Set Credentials*. All commands and URLs throughout the workshop guide will update automatically to use your values.

[TIP]
====
Once set, your credentials are preserved as you navigate between lab pages.
If you see `{user}` in any command, click *Change* in the credential bar and re-enter your details.
====

Click the *Web Console* tab in the right panel of this page to open the embedded OpenShift web console.

Log in with the credentials provided by your instructor:

* *Username:* `{user}`
* *Password:* `{password}`

== Step 4: Verify Namespace Access

You have six namespaces assigned to you, one per module:

* `{ns_configuration}` -- Module 1: Configuration labs
* `{ns_storage}` -- Module 2: Storage labs
* `{ns_security}` -- Module 3: Security labs
* `{ns_networking}` -- Module 4: Networking labs
* `{ns_resources}` -- Module 5: Resources labs
* `{ns_lifecycle}` -- Module 6: Lifecycle labs

Verify you can list your namespaces:

[.console-input]
[source,bash,subs="attributes+"]
----
oc get projects | grep {user}
----

You should see all six namespaces listed. Each namespace contains broken workloads for that module's lab exercises.

Verify access to one namespace as an example:

[.console-input]
[source,bash,subs="attributes+"]
----
oc get pods -n {ns_configuration}
----

You should see several pods in various error states -- that is expected! Each pod corresponds to a lab exercise.

== Step 5: Explore the Web Console

In the OpenShift Web Console:

. Switch to the *Developer* perspective using the dropdown in the top-left
. Select your namespace `{ns_configuration}` from the *Project* dropdown
. Click *Topology* to see the deployed applications
. Notice the various error indicators on the application icons

== Tips for the Workshop

* *Use both the Web Console and CLI* -- the labs provide instructions for both approaches
* *Read error messages carefully* -- OpenShift error messages are descriptive and point you in the right direction
* *Use the hints progressively* -- try to solve each lab before opening hints
* *Check the xref:99-cheatsheet.adoc[oc Cheat Sheet]* for command reference
* *Ask for help* -- your instructor is available if you get stuck
