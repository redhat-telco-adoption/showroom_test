== Overview

The `lab-4-4-frontend` application cannot reach the `lab-4-4-backend` service. Both pods are running, DNS resolves correctly, but direct HTTP connections from frontend to backend time out. A NetworkPolicy on the backend only allows ingress from pods with the label `role: api-gateway`, but the frontend has `role: frontend`. Your mission is to fix the NetworkPolicy to allow the frontend to reach the backend.

== Hints

[%collapsible]
.Hint 1 -- Where to look
====
NetworkPolicies use label selectors to determine which pods can communicate. Check the ingress rules on the backend's NetworkPolicy and compare them with the frontend pod's labels.
====

[%collapsible]
.Hint 2 -- What's wrong
====
The backend's NetworkPolicy `lab-4-4-backend-ingress` allows ingress only from pods with `role: api-gateway`. The frontend pods have `role: frontend`. Either update the NetworkPolicy or change the frontend label.
====

[%collapsible]
.Hint 3 -- Almost there
====
The recommended fix is to update the NetworkPolicy's `podSelector` in the ingress `from` section to match `role: frontend`. This is more correct than relabeling the frontend pods.
====

== Fix the Issue

Update the backend's NetworkPolicy to allow ingress traffic from the frontend pods. Modify the allowed label selector to match the frontend's actual labels.

[%collapsible]
.Reveal Solution (Instructor Reference Only)
====
Edit the NetworkPolicy:

[.console-input]
[source,bash,subs="attributes+"]
----
oc edit networkpolicy lab-4-4-backend-ingress -n {module_namespace}
----

Change the ingress `from` selector from:

[source,yaml]
----
from:
- podSelector:
    matchLabels:
      role: api-gateway
----

To:

[source,yaml]
----
from:
- podSelector:
    matchLabels:
      role: frontend
----

Or use patch:

[.console-input]
[source,bash,subs="attributes+"]
----
oc patch networkpolicy lab-4-4-backend-ingress -n {module_namespace} --type=json \
  -p='[{"op": "replace", "path": "/spec/ingress/0/from/0/podSelector/matchLabels/role", "value": "frontend"}]'
----
====
