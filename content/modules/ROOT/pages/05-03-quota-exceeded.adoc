== Overview

The Deployment `lab-5-3-app` is configured with 3 replicas, but not all pods are starting. The namespace has a ResourceQuota that limits the total number of pods and memory. The Deployment exceeds these quota limits. Your mission is to adjust the Deployment to fit within the quota constraints.

== Hints

[%collapsible]
.Hint 1 -- Where to look
====
A ResourceQuota in the namespace limits the total number of pods and the total memory that can be requested. The Deployment wants 3 replicas at 256Mi each, which exceeds both the pod count limit (2) and memory limit (512Mi).
====

[%collapsible]
.Hint 2 -- What needs to change
====
You need to reduce either the number of replicas, the memory per replica, or both. The quota allows a maximum of 2 pods and 512Mi total memory. Note that other lab pods in the namespace also count toward the quota.
====

[%collapsible]
.Hint 3 -- Almost there
====
Scale down to 2 replicas and reduce memory requests to `200Mi` each. This keeps you within the quota: 2 pods and 400Mi total (under the 512Mi limit), leaving room for other lab pods.
====

== Fix the Issue

Adjust the Deployment's replica count and memory requests to fit within the namespace ResourceQuota.

[%collapsible]
.Reveal Solution (Instructor Reference Only)
====
Scale down to 2 replicas:

[.console-input]
[source,bash,subs="attributes+"]
----
oc scale deployment lab-5-3-app --replicas=2 -n {module_namespace}
----

Reduce memory requests:

[.console-input]
[source,bash,subs="attributes+"]
----
oc set resources deployment/lab-5-3-app \
  --requests=memory=200Mi \
  --limits=memory=200Mi \
  -n {module_namespace}
----
====
