[.label.label-green]#Beginner# | *Persona:* Developer | *Category:* Configuration & Resources

== Overview

The application `lab-1-3-app` is crashing immediately after startup. The container logs reveal that required environment variables are missing. Your mission is to examine the logs, determine what environment variables the application needs, and add them to the Deployment.

== Hints

[%collapsible]
.Hint 1 -- Where to look
====
The application startup script checks for environment variables before starting. The logs tell you exactly which variable is missing. Read the error message carefully.
====

[%collapsible]
.Hint 2 -- What's missing
====
The application requires two environment variables: `DATABASE_URL` and `APP_NAME`. You can set `DATABASE_URL` to any valid connection string (e.g., `postgresql://localhost:5432/mydb`) and `APP_NAME` to any application name.
====

[%collapsible]
.Hint 3 -- Almost there
====
Use `oc set env` to add the missing environment variables to the Deployment, or edit the Deployment YAML directly to add them under `spec.template.spec.containers[0].env`.
====

== Fix the Issue

Add the required environment variables to the `lab-1-3-app` Deployment. The application needs `DATABASE_URL` and `APP_NAME` to be set.

[%collapsible]
.Reveal Solution (Instructor Reference Only)
====
[.console-input]
[source,bash,subs="attributes+"]
----
oc set env deployment/lab-1-3-app \
  DATABASE_URL=postgresql://localhost:5432/mydb \
  APP_NAME="Troubleshooting Workshop" \
  -n {user}-lab
----

This triggers a new rollout. The pod should start successfully with the environment variables set.
====
