== Prerequisites

Before attending this workshop you should be comfortable with:

* Basic Linux command-line usage (navigating directories, running commands)
* Fundamental Kubernetes or OpenShift concepts: Pods, Deployments, Services, and namespaces
* Familiarity with YAML syntax

No prior OpenShift troubleshooting experience is required.

== Environment access

Your workshop environment consists of:

* A dedicated OpenShift cluster provisioned for this session
* A Showroom interface with embedded Web Console and terminal
* Pre-deployed broken applications ready for you to fix

Access credentials (username, password, and cluster subdomain) are provided by your instructor. Enter them in the credential bar at the top of this page. All commands and URLs in the labs will update automatically to use your values.

== Tools used in this workshop

=== oc CLI

All labs include step-by-step `oc` CLI instructions. The `oc` command-line tool is the primary interface for interacting with OpenShift. It is pre-installed in the embedded terminal.

Key commands used throughout the workshop include:

* `oc get pods` -- list pod status
* `oc describe pod` -- detailed pod info and events
* `oc logs` -- container log output
* `oc get events` -- namespace event stream
* `oc edit` / `oc patch` / `oc set` -- make changes to resources

See the xref:99-cheatsheet.adoc[oc Cheat Sheet] for a full reference.

=== OpenShift Web Console

Each lab also provides Web Console instructions as an alternative to the CLI. The Web Console is embedded in the right panel of the Showroom interface.

=== OpenShift version

This workshop targets OpenShift 4.12 or later. All commands and features shown are available in current OpenShift 4.x releases.
