== Module breakdown

This workshop consists of 7 hands-on modules plus a getting-started orientation. Each module addresses a specific category of OpenShift failure.

=== Full workshop (6 hours)

[cols="1,3,1",options="header"]
|===
|Module |Topic |Time

|Module 1
|Getting started: CLI, web console, and diagnostic tools
|30 min

|Module 2
|Configuration errors: Missing ConfigMaps, Secrets, and invalid environment variables
|60 min

|Module 3
|Storage issues: Missing PVCs, pending volumes, and permission errors
|60 min

|Module 4
|Security and access: RBAC errors, SCC violations, and image pull secrets
|60 min

|Module 5
|Networking problems: Selector mismatches, route issues, DNS failures, and network policy
|75 min

|Module 6
|Resource constraints: Insufficient resources, OOMKilled pods, and quota exceeded errors
|60 min

|Module 7
|Application failures: CrashLoopBackOff, probe failures, and wrong image tags
|60 min

|Conclusion
|Cheatsheet, patterns reference, and next steps
|15 min
|===

=== Abbreviated workshop (2-3 hours)

If you have limited time, focus on these high-value modules:

* *Module 2*: Configuration errors (most common issue category for new OpenShift users)
* *Module 5*: Networking problems (second most common issue category)
* *Module 7*: Application failures (CrashLoopBackOff is a frequent support request)

== Technical requirements

=== Software versions

* Red Hat OpenShift {ocp_version} or later
* OpenShift CLI (`oc`) compatible with {ocp_version}
* Web browser (Chrome, Firefox, or Edge recommended)

=== Environment access

Your workshop environment includes:

* A dedicated OpenShift {ocp_version} multinode cluster
* Pre-provisioned namespaces with broken deployments ready to investigate in each module
* Web terminal access via the OpenShift console (no local `oc` install required)
* Your cluster credentials shown in the lab interface

No local software installation is required. All exercises use the web terminal embedded in the OpenShift console.

== Environment setup

Your workshop environment has been pre-configured with:

* All required namespaces and broken deployments
* A web terminal already authenticated to the cluster
* The `oc` CLI available in the web terminal

=== Pre-workshop checklist

Run these commands in the web terminal to verify your setup:

[source,bash]
----
# Verify CLI access
oc whoami

# Verify cluster version
oc version --short

# List available namespaces
oc get projects | grep acme
----

Expected output for `oc whoami`:

----
{bastion_ssh_user_name}
----

Expected output for `oc version --short`:

----
Client Version: 4.17.x
Server Version: 4.17.x
Kubernetes Version: v1.30.x
----

If these commands fail, refer to the troubleshooting section in xref:03-module-01-getting-started.adoc[Module 1].
