= Lab 3.1: RBAC Configuration
include::_attributes.adoc[]

[.label.label-yellow]#Intermediate# | *Persona:* Platform Engineer | *Category:* Security & Access Control

== Overview

In the `{user}-admin-lab` namespace, a Job runs under the `deploy-bot` ServiceAccount and attempts to create a Deployment. The Job fails because the ServiceAccount lacks the necessary RBAC permissions. Your mission is to diagnose the RBAC denial and create the appropriate Role and RoleBinding so the Job can succeed.

== Hints

[%collapsible]
.Hint 1 -- Where to look
====
The `deploy-bot` ServiceAccount exists but has no Roles or RoleBindings granting it permission to manage Deployments. You need to create both a Role and a RoleBinding.
====

[%collapsible]
.Hint 2 -- What permissions are needed
====
The Job needs to create, get, list, and update Deployments in the `apps` API group. Create a Role with these verbs on the `deployments` resource.
====

[%collapsible]
.Hint 3 -- Almost there
====
Create a Role with `oc create role` and then bind it with `oc create rolebinding`. The RoleBinding should reference the `deploy-bot` ServiceAccount.
====

== Fix the Issue

Create a Role granting deployment management permissions and bind it to the `deploy-bot` ServiceAccount in the `{user}-admin-lab` namespace. Then re-run the Job.

[%collapsible]
.Reveal Solution (Instructor Reference Only)
====
Create the Role:

[.console-input]
[source,bash,subs="attributes+"]
----
oc create role deployer \
  --verb=create,get,list,update \
  --resource=deployments.apps \
  -n {user}-admin-lab
----

Create the RoleBinding:

[.console-input]
[source,bash,subs="attributes+"]
----
oc create rolebinding deployer-binding \
  --role=deployer \
  --serviceaccount={user}-admin-lab:deploy-bot \
  -n {user}-admin-lab
----

Delete the failed Job and recreate it:

[.console-input]
[source,bash,subs="attributes+"]
----
oc delete job lab-3-1-deployer-job -n {user}-admin-lab
oc create job lab-3-1-deployer-job \
  --from=cronjob/lab-3-1-deployer-job \
  -n {user}-admin-lab 2>/dev/null || \
oc get job lab-3-1-deployer-job -n {user}-admin-lab -o yaml | \
  oc replace --force -f -
----

If the above does not work, apply the Job manifest again from the install directory.
====
