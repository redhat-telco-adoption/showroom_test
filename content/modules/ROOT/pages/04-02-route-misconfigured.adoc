== Overview

The application `lab-4-2-app` is running and the Service has correct endpoints, but the Route still returns a `503 Service Unavailable` error. The Route's `targetPort` does not match the port exposed by the Service. Your mission is to find the port mismatch and correct it.

== Learning objectives

* Identify a `503 Service Unavailable` error caused by a Route targeting a non-existent Service port
* Understand how Route `targetPort` must match an actual port exposed by the Service
* Apply a Route patch to correct the `targetPort` and restore application access

====

=== Hints

[%collapsible]
.Hint 1 -- Where to look
====
Compare the Route's `targetPort` with the Service's available ports. The Route must reference a port that the Service actually exposes.
====

[%collapsible]
.Hint 2 -- What's wrong
====
The Route has `targetPort: 8443` but the Service only exposes port `8080`. The Route cannot forward traffic to a port the Service does not have.
====

[%collapsible]
.Hint 3 -- Almost there
====
Edit the Route and change `targetPort` from `8443` to `8080`. Use `oc edit route` or `oc patch`.
====

== Exercise 2: Resolve the issue

=== Fix the issue

Update the Route's `targetPort` to match the port exposed by the Service.

[%collapsible]
.Reveal Solution (Instructor Reference Only)
====
[.console-input]
[source,bash,subs="attributes+"]
----
oc patch route lab-4-2-route -n {module_namespace} \
  --type=json -p='[{"op": "replace", "path": "/spec/port/targetPort", "value": "8080"}]'
----
====

=== Verify your fix
