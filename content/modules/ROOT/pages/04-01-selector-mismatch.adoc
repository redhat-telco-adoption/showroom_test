== Overview

The application `lab-4-1-app` pods are running and healthy, but accessing the Route returns a `503 Service Unavailable` error. The Service is not routing traffic to the pods because of a label mismatch between the Service selector and the pod labels. Your mission is to find and fix the mismatch.

== Diagnose

[tabs]
====
Web Console::
+
--
. Navigate to *Networking -> Services* in namespace `{module_namespace}`
. Click `lab-4-1-svc` and note the *Selector* field (e.g., `app=myapp`)
. Navigate to *Workloads -> Pods* and click on the running pod
. Check the pod's *Labels* -- notice the label is `app=my-app` (with a hyphen)
. The selector `app=myapp` does not match the label `app=my-app`
--

Web Terminal (`oc`)::
+
--

== Hints

[%collapsible]
.Hint 1 -- Where to look
====
The Service has a `selector` that determines which pods receive traffic. Compare the Service's selector with the actual labels on the pods. Use `oc get endpoints` to verify whether the Service finds any matching pods.
====

[%collapsible]
.Hint 2 -- What's wrong
====
The Service selector is `app: myapp` but the pods have the label `app: my-app`. The hyphen makes the difference -- the selector must exactly match the pod labels.
====

[%collapsible]
.Hint 3 -- Almost there
====
Edit the Service to change its selector from `app: myapp` to `app: my-app` to match the pod labels. Use `oc edit svc` or `oc patch`.
====

== Fix the Issue

Update the Service selector to match the actual pod labels so that endpoints are populated and traffic flows correctly.

[%collapsible]
.Reveal Solution (Instructor Reference Only)
====
[.console-input]
[source,bash,subs="attributes+"]
----
oc patch svc lab-4-1-svc -n {module_namespace} \
  --type=json -p='[{"op": "replace", "path": "/spec/selector/app", "value": "my-app"}]'
----

Alternatively:

[.console-input]
[source,bash,subs="attributes+"]
----
oc edit svc lab-4-1-svc -n {module_namespace}
----

Change `app: myapp` to `app: my-app` under `spec.selector`.
====
