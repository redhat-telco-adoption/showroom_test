= Lab 1.2: Missing Secret
include::_attributes.adoc[]
:module_namespace: {ns_configuration}

[.label.label-green]#Beginner# | *Persona:* Developer | *Category:* Configuration & Resources

== Overview

The application `lab-1-2-app` cannot start because it references a Secret that does not exist. The container needs database credentials injected as environment variables, but the Secret has not been created. Your mission is to find out which Secret is missing and create it with the required keys.

== Learning objectives

* Identify `CreateContainerConfigError` caused by a missing Secret using `oc describe pod` and events
* Understand how Secrets inject environment variables and why missing Secrets block container startup
* Apply `oc create secret generic` with the required keys to restore the application to `Running` state

=== Hints

[%collapsible]
.Hint 1 -- Where to look
====
The Deployment uses `envFrom` or `env` with `secretKeyRef` to inject environment variables from a Secret named `db-credentials`. Inspect the Deployment spec to see which keys are expected.
====

[%collapsible]
.Hint 2 -- What keys are needed
====
The Deployment expects 3 keys from the Secret: `MYSQL_USER`, `MYSQL_PASSWORD`, and `MYSQL_DATABASE`. All 3 must be present.
====

[%collapsible]
.Hint 3 -- Almost there
====
Create a Secret with `oc create secret generic db-credentials` and include all 3 required key-value pairs. The values can be any valid strings for this lab.
====

== Exercise 2: Resolve the issue

=== Fix the issue

Create the missing Secret named `db-credentials` in the `{module_namespace}` namespace. It must contain the 3 keys that the Deployment expects as environment variables.

[%collapsible]
.Reveal Solution (Instructor Reference Only)
====
[.console-input]
[source,bash,subs="attributes+"]
----
oc create secret generic db-credentials \
  --from-literal=MYSQL_USER=appuser \
  --from-literal=MYSQL_PASSWORD=s3cur3pass \
  --from-literal=MYSQL_DATABASE=workshop_db \
  -n {module_namespace}
----

After creating the Secret, delete the existing pod to trigger a new one:

[.console-input]
[source,bash,subs="attributes+"]
----
oc delete pod -l app=lab-1-2-app -n {module_namespace}
----

The new pod should start successfully.
====
