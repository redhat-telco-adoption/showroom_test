= Lab 2.1: Missing PVC
include::_attributes.adoc[]
:module_namespace: {ns_storage}

[.label.label-green]#Beginner# | *Persona:* Developer | *Category:* Storage

== Overview

The application `lab-2-1-app` is stuck in `Pending` state. The pod cannot be scheduled because it references a PersistentVolumeClaim that does not exist. Your mission is to create the missing PVC so the pod can be scheduled and start successfully.

== Learning objectives

* Identify a `Pending` pod caused by a missing PersistentVolumeClaim using `oc describe pod` and events
* Understand how pod scheduling fails when referenced PVCs do not exist
* Apply a PVC manifest to create the missing storage resource and allow the pod to be scheduled

=== Hints

[%collapsible]
.Hint 1 -- Where to look
====
The Deployment references a PVC named `data-volume` in its volumes section. This PVC needs to be created before the pod can be scheduled.
====

[%collapsible]
.Hint 2 -- What to create
====
You need to create a PersistentVolumeClaim named `data-volume` with `ReadWriteOnce` access mode and a reasonable size (e.g., `1Gi`). Use the cluster's default StorageClass.
====

[%collapsible]
.Hint 3 -- Almost there
====
Create the PVC with `oc apply -f` using a YAML manifest, or use `oc create` to create it imperatively. You do not need to specify a StorageClass -- the cluster default will be used.
====

== Exercise 2: Resolve the issue

=== Fix the issue

Create the missing PersistentVolumeClaim named `data-volume` in the `{module_namespace}` namespace. It should use `ReadWriteOnce` access mode.

[%collapsible]
.Reveal Solution (Instructor Reference Only)
====
[.console-input]
[source,bash,subs="attributes+"]
----
cat <<'EOF' | oc apply -n {module_namespace} -f -
apiVersion: v1
kind: PersistentVolumeClaim
metadata:
  name: data-volume
spec:
  accessModes:
    - ReadWriteOnce
  resources:
    requests:
      storage: 1Gi
EOF
----

The PVC will bind to an available PV, and the pod will transition from `Pending` to `Running`.
====

=== Verify your fix

Check that the PVC is bound:

[.console-input]
[source,bash,subs="attributes+"]

----
